# Spike Creations — Tracker (single-file backend)

Everything the server needs — data, storage, login, and every API route —
now lives in **one file: `server.js`**. The only other file you'll ever
need to touch is **`public/index.html`** (the page itself).

This means future updates are simple: replace `server.js` and/or
`public/index.html`, then commit and push. No more `src/` folder, no more
`src/routes/` subfolder, no more mixing up where a file belongs.

## Folder structure

```
spike-tracker/
├── server.js          <- ALL backend code (data, auth, storage, API routes)
├── public/
│   └── index.html     <- the whole page (frontend)
├── package.json
├── .env.example
├── .gitignore
├── data/               <- created automatically, holds db.json
└── uploads/            <- created automatically, holds uploaded files
```

## Running it locally

```bash
npm install
cp .env.example .env     # edit JWT_SECRET before deploying anywhere real
npm start                 # http://localhost:4000
```

To wipe all data back to the original seed accounts/entries:
```bash
npm run seed:reset
```
(then restart the server — it reseeds automatically)

## Downloading entries (Word / Excel)

- **Any entry card** (Factory Feed, My Dept, or a Section) has ⬇ Word and
  ⬇ Excel buttons at the bottom — same view rules as seeing that entry in
  the first place, no extra permissions needed.
- **Admin / MD Dashboard** has a "Full Company Report" card with one-click
  Word and Excel downloads covering *every* department and section in a
  single file (Excel: one sheet per section + a summary sheet; Word: one
  heading + table per section). This is admin/MD only.
- Generated by `exceljs` and `docx` — both were just added to
  `package.json`, so run `npm install` again after pulling this update.

## Making future updates

If you want to add/change a department, section, or field yourself later:

1. Open `server.js`, find the `SECTIONS` (or `DEPARTMENTS`) array near the
   top, and edit it.
2. Open `public/index.html`, find the **same** `SECTIONS` array (search for
   `const SECTIONS = [`), and make the **identical** edit there. The page
   keeps its own copy to render the forms — the two must always match.
3. Test locally (`npm start`, check `http://localhost:4000`).
4. Copy both files into your GitHub folder, commit, and push in GitHub
   Desktop. Render redeploys automatically.

Or — simpler — just tell me what you want changed and I'll edit both
files for you; you'll just need to copy them over and push.

## Deploying / going live

Same as before: push to GitHub, connect the repo to a Render Web Service
with Build Command `npm install` and Start Command `npm start`, and add
an environment variable `JWT_SECRET` with any random value.

**Free-tier note:** Render's free plan wipes `data/` and `uploads/` on
redeploy, so entries and attached files won't survive long-term on the
free plan. For real day-to-day team use, either add a small paid
persistent disk, or ask and I can switch storage to a proper hosted
database.
